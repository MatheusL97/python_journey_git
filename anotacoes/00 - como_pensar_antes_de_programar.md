# 🐍 Python Journey

# 00 - Como Pensar Antes de Programar

## 🎯 Objetivo

Aprender a organizar o raciocínio antes de escrever qualquer linha de código.

---

# 📖 Introdução

Um dos erros mais comuns de quem está começando a programar é abrir o editor de código e começar a escrever sem antes entender o problema.

Programar não é apenas escrever comandos em Python.

Programar é resolver problemas.

O código é apenas a ferramenta utilizada para transformar uma solução em algo que o computador consiga executar.

Por isso, antes de abrir o VS Code, devemos responder quatro perguntas.

---

# 🧠 Os 4 Passos do Pensamento

## 1️⃣ Qual é o problema?

Entenda exatamente o que precisa ser resolvido.

Exemplo:

> Criar um programa que informe se uma pessoa pode votar.

---

## 2️⃣ Quais informações preciso receber?

Liste apenas os dados necessários.

Exemplo:

- Idade

---

## 3️⃣ O que o programa precisa fazer?

Descreva, com suas palavras, o processamento.

Exemplo:

- Verificar se a idade é suficiente para votar.

---

## 4️⃣ O que o programa deve mostrar?

Defina qual será a saída para o usuário.

Exemplo:

- "Você pode votar."
- "Você não pode votar."

---

# 📌 Modelo IPO

Todo problema pode ser dividido em três partes.

```text
INPUT
⬇
PROCESS
⬇
OUTPUT
```

### Input (Entrada)

São as informações fornecidas pelo usuário.

Exemplo:

- Nome
- Idade
- Nota

---

### Process (Processamento)

É o que o programa faz com essas informações.

Exemplo:

- Somar valores
- Comparar números
- Calcular média
- Verificar condições

---

### Output (Saída)

É o resultado apresentado ao usuário.

Exemplo:

- Média do aluno
- Valor total
- Situação do aluno

---

# 💡 Exemplo

Problema:

Descobrir se uma pessoa pode votar.

### Input

- Idade

### Process

- Verificar se a idade atende ao requisito para votar.

### Output

- "Você pode votar."
- "Você não pode votar."

---

# ⚠️ Erro mais comum

Começar a programar antes de entender o problema.

Isso normalmente resulta em:

- Código confuso
- Muitas correções
- Dificuldade para encontrar erros

---

# 🧠 Como pensar como um desenvolvedor

Sempre faça estas perguntas antes de começar:

- Qual problema preciso resolver?
- Quais informações preciso?
- O que deve acontecer?
- Qual resultado quero mostrar?

Responder essas perguntas torna a programação muito mais simples.

---

# 📌 Resumo

✔ Todo programa resolve um problema.

✔ Antes de programar, entenda o problema.

✔ Organize o raciocínio usando o modelo IPO.

✔ Só depois comece a escrever o código.
